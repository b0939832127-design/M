# 心情養成寵物日記 (Mood Pet Diary) — 專案規格說明書 (SPEC.md)

本專案為一個結合**心情日記記錄**與**虛擬寵物養成**的身心療癒 Web 應用程式。使用者可以透過每天記錄心情與生活點滴，為虛擬寵物提供成長能量，解鎖互動對話與情緒分析，藉此建立健康的心情調適習慣。

---

## 1. 專案技術棧與系統架構

### 前端技術
- **HTML5**: 提供語意化的網頁骨架結構。
- **CSS3 (Vanilla CSS)**: 使用現代 CSS 變數（CSS Tokens），並實作 **Glassmorphism（磨砂玻璃風格）** 設計系統，搭配細緻的微動畫與漸層效果。
- **JavaScript (Vanilla JS)**: 處理前端的核心邏輯、狀態管理、DOM 操作以及與模擬資料庫的互動。包含獨立的 AI 設定與 API 對接模組 [ai-config.js](file:///c:/Coding/New%20project/M/ai-config.js)。
- **Google Fonts**: 引用了 `Nunito`（主要英文字型）與 `Noto Sans TC`（主要中文字型），營造溫暖、親和的視覺風格。
- **Chart.js (CDN)**: 用於情緒歷史追蹤頁面的圖表繪製。
- **Groq API**: 整合 `llama-3.3-70b-versatile` 大型語言模型，提供即時、情緒感知的虛擬寵物聊功能。

### 資料儲存
- **Local Storage**: 
  - **應用程式資料庫**：Key 值為 `petDiaryDB_v1`，用於儲存使用者的日記列表及虛擬寵物的所有成長數值。
  - **API 金鑰安全儲存**：Key 值為 `groq_api_key`，將使用者輸入的 API Key 本地留存，避免伺服器端暴露。

## 2. 資料模型與結構 (Data Schema)

在 `db.js` 中定義的 `PetDB` 資料庫結構如下：

```json
{
  "pet": {
    "name": "小布丁",
    "level": 1,
    "exp": 20,
    "intimacy": 50,
    "streak": 0
  },
  "diaries": [
    {
      "date": "2026/5/30",
      "mood": "喜",
      "moodVal": 4,
      "text": "今天和朋友一起去野餐，天氣非常好！",
      "photo": "data:image/png;base64,...",
      "timestamp": 1780111200000
    }
  ]
}
```

### 欄位說明

#### 1. 寵物狀態 (`pet`)
- `name` (String): 寵物名稱，預設為 `"小布丁"`。
- `level` (Number): 寵物等級，初始為 `1`。
- `exp` (Number): 經驗值 (0 - 100 XP)，初始為 `20`。滿 100 時會升級並扣除 100 XP。
- `intimacy` (Number): 親密度 (0 - 100)，初始為 `50`。
- `streak` (Number): 連續記錄日記天數，初始為 `0`。

#### 2. 日記結構 (`diaries[]`)
- `date` (String): 日記日期格式（例如 `"2026/5/30"`）。
- `mood` (String): 心情分類，共有 `喜`、`樂`、`怒`、`哀` 四種。
- `moodVal` (Number): 心情指數值（1 至 4），用以繪製圖表：
  - `喜` = `4`
  - `樂` = `3`
  - `怒` = `2`
  - `哀` = `1`
- `text` (String): 日記內文。
- `photo` (String | null): 使用者上傳的圖片編碼（Base64 DataURL格式，選填）。
- `timestamp` (Number): 記錄存檔時的時間戳記（毫秒）。

## 3. 功能模組規格

本專案共由四個主要 HTML 頁面組成，各網頁之功能規格如下：

### 🏠 寵物主頁 ([index.html](file:///c:/Coding/New%20project/M/index.html))
主頁為使用者登入後的核心控制面板，展示寵物的當前狀況與最近的心情摘要。
1. **寵物成長狀態面板**：
   - 讀取並展示寵物的「等級」、「經驗值百分比進度條 (0-100 XP)」、「親密度百分比進度條 (0-100)」及「日記總篇數」。
   - 展示「連續記錄天數」的火焰計數器。
   - **客製化寵物姓名**：名字旁提供 ✏️ 按鈕。使用者點擊後會彈出輸入框，允許將寵物名稱修改並儲存於 `localStorage`（限制 1-10 個字元），頁面上所有相關顯示（包含日記頁的 placeholder、alt 屬性與 Modal 提示）皆會動態同步更新。
2. **動態對話泡泡 (Speech Bubble)**：
   - 系統會讀取最新一筆日記的心情，讓小布丁說出對應的互動台詞：
     - **喜**：*「今天你心情不錯呢！謝謝你陪著我😊」*
     - **樂**：*「哇！感受到你的快樂了！一起跳舞吧！😆」*
     - **怒**：*「發生什麼不開心的事嗎？我在這裡陪你😤」*
     - **哀**：*「難過的時候也要記得我喔…抱抱🥺」*
     - 若無歷史日記，則顯示預設歡迎詞：*「嗨！今天心情好嗎？」*
3. **最近日記預覽**：
   - 列出最近的 3 篇日記，顯示其情緒 Icon、日期與簡短內文。
   - 若尚無日記，則顯示空狀態 (Empty State) 卡片，引導使用者點擊「開始記錄」按鈕。
4. **AI 即時聊天對話區 (Chat Section)**：
   - **Groq API 金鑰管理**：提供摺疊面板輸入並安全儲存 API Key 至 LocalStorage，若無 Key 則顯示系統提示引導設定。
   - **動態人設 System Prompt**：AI 寵物回覆會依據當前的「寵物名字」、「寵物等級」、「親密度數值（高等級更撒嬌、低等級害羞）」以及「最近日記的心情狀態（進行情緒安撫或共享歡樂）」動態調整說話語氣與人設。
   - **聊天對話流**：支援 Enter 鍵或點擊傳送，有打字中動畫（Typing Indicator）以及自動滾動。對話歷史僅保留最近 3 輪（6 條訊息）作為短期記憶。
   - **AI 設定獨立**：核心 AI 設定與對接封裝於獨立的 [ai-config.js](file:///c:/Coding/New%20project/M/ai-config.js) 中，使用 `llama-3.3-70b-versatile` 模型。

---

### 📝 記錄日記頁 ([diary.html](file:///c:/Coding/New%20project/M/diary.html))
提供使用者撰寫日記、抒發心情並與寵物即時互動的表單頁面。
1. **心情選擇器**：
   - 提供 4 個直覺的表情按鈕：**喜 (😊)**、**樂 (😆)**、**怒 (😠)**、**哀 (😢)**。
   - 點擊按鈕時觸發選取狀態（高亮與陰影動畫），並即時更新：
     - 右側寵物的對話泡泡（提供溫暖的回饋文字）。
     - 右側顯示對應情緒的「💡 今日小提示」（提供情緒調適、正向思考的文字建議）。
2. **文字日記區**：
   - 多行文字輸入框，並有即時的**字數統計器**。
3. **照片上傳區**：
   - 支援點擊或拖曳（Drag and Drop）上傳檔案。
   - 使用 `FileReader` 將圖片即時讀取並轉換為 Base64 DataURL，並顯示即時圖片預覽。
4. **儲存機制與數值提升**：
   - 當使用者點擊「儲存今日日記」：
     - 若未選擇心情，會以 `alert` 提示。
     - 儲存時，寵物的**親密度 +8** (上限 100)，**經驗值 +20**。
     - 當經驗值滿 100 XP 時，寵物**等級 (Level) +1**，且經驗值歸零或扣除 100。
     - **連續紀錄天數 (Streak)**：交由系統基於日記時間戳記動態計算（同一天記錄多篇不重複累計，中斷時自動校正）。
     - 將資料寫入 LocalStorage。
5. **成功提示彈窗 (Modal)**：
   - 日記成功儲存後，彈出毛玻璃風格的 Modal，顯示寵物跳躍動畫與更新的親密度/等級數值。
   - 提供「回到主頁」與「繼續紀錄」的雙導向按鈕。

### 📈 情緒圖表頁 ([analytics.html](file:///c:/Coding/New%20project/M/analytics.html))
透過圖表將使用者的心情紀錄視覺化，協助使用者追蹤長期的心理健康趨勢。
1. **情緒摘要卡片**：
   - **總日記篇數**：顯示目前累計寫下的日記數量。
   - **最常見心情**：統計出現頻率最高的心情，並顯示對應的表情符號。
   - **平均心情指數**：計算所有日記心情數值的平均值，並對應到最接近的情緒標籤。
   - **連續記錄天數**：同步展示當前的連續天數。
2. **心情趨勢圖 (Chart.js 折線圖)**：
   - 將日期作為 X 軸，心情指數（1-4）作為 Y 軸。
   - Y 軸刻度以中文字「哀、怒、樂、喜」展示，取代生硬的數字。
   - 圖表控制：可切換查看「最近 7 天」、「最近 30 天」與「全部」的情緒走勢。
3. **心情分佈圖 (Chart.js 甜甜圈圖)**：
   - 統計四種情緒的比例分佈，以精美的配色（紫、綠、橘、藍）區分不同心情。
4. **全部日記歷史紀錄**：
   - 以時間倒序（最新在最前）列出所有歷史日記。
   - 每筆紀錄包含心情表情、日期、日記內文，若有上傳照片亦會在此展示縮圖。

### 🗺️ 旅程地圖展示頁 ([journey.html](file:///c:/Coding/New%20project/M/journey.html))
這是一個展示本產品「使用者體驗設計 (UX Design)」研究成果的特色頁面。
1. **使用者角色誌 (Personas)**：
   - **角色 A：上班族小明**（面臨朝九晚五的重複工作壓力與社交疲勞）。
   - **角色 B：研究生小晴**（面對論文與實驗的高壓焦慮）。
   - 提供互動切換，展現不同角色在使用產品時的痛點。
2. **Hook 核心模型模擬 (Hook Model)**：
   - 詳細拆解使用者的習慣養成路徑：
     - **Trigger (外部與內部觸發)**：如傍晚的系統推播、下班後的空虛感。
     - **Action (行動)**：點擊 App、選擇今日情緒按鈕並寫下簡短文字。
     - **Reward (多變的酬賞)**：寵物溫馨的擬人化回饋、親密度與經驗值成長的視覺回饋。
     - **Investment (投入)**：日積月累的心情歷史圖表、親密的寵物依絆，增加使用者流失的沉沒成本。
3. **情緒與壓力波動曲線 (互動式 SVG 圖表)**：
   - 模擬使用者在使用 App 前後的心情、壓力和依絆度走勢，展示產品對使用者心理健康的長期正向影響。

為了未來能將此靜態應用程式升級為前後端分離的系統（預計採用 Flask 後端），`db.js` 中已預留了 API 對接規範。

### API 路由規格表

| 請求方法 | 路由路徑 | 功能說明 | 負責人員 / 功能編號 |
| :--- | :--- | :--- | :--- |
| **GET** | `/api/diary` | 讀取目前使用者所有的日記列表與寵物成長數值 | — |
| **POST** | `/api/diary` | 新增一筆心情日記，並同步更新後端寵物的親密度與經驗值 | 廖翾端 (F-01) |
| **GET** | `/api/pet` | 單獨讀取虛擬寵物的各項成長數值 (Level, EXP, Intimacy 等) | 林祐群、王柔閔 (F-02) |
